Resources and tools

HTML5:
HTML5 is a markup languages used for web development. It is responsible providing a structure and presenting contents on the web. We're using html5 as it's one of the most common languages for web developement and we all have some experience using it, it is also free. There isn't really an alternative to html as majority of browser used html, meaning html is pretty much required.


CSS3:
CSS is a stylesheet language thats compatible with html5. It is used to deal with presentation of a web page, essentially it is used to make a web page look pretty. This is going to be used for our web design as there isn't an alternative for CSS.


PHP:
PHP is a server scripting language for web development. This is going to be used to handle account management in our project such as registration and login. It is free and it can work with different database language such as MySQL. Alternatives are Javascript/Node.js.

Alt: Node.js

mySQL:
It's a opensource database management Systems. This is going to be used for databse management such as storing accouint information and it also works with PHP.

Alt: MySQL

Collaborative Workspace

Google docs:
All team members are able to access a shared google doc to edit Assignment 1. A link to join is accessible through teams.

Trello:
Trello allows for different tasks to easily be managed and accessed by group members on cards. Members can add and edit tasks and communicate through comments. Trello board has been implemented on teams so all team members can view the board through teams without needing to view an external app.

GitHub:
GitHub allows for file sharing and version management between members. Every member is able to upload files to GitHub to be previewed or downloaded. A link to access GitHub is available to all group members on teams.

Group Contract

Expectations:

  • Team members are expected to be present for all team meeting which occur at Friday, 4.00pm. If a team member can be present, the team must be notified via Microsoft Teams.
  • Workload of a team member is to be a 1 hour/day minimum, 2 hours/day is the optimal amount.
  • Team members are treated as equals and must be respectful of each other.
  • Team members are expected to update their Trello cards.
  • If a problem or disagreement is instigated, a team meeting must occur and is to be resolved before any progress of the project occurs.

Communication:

Microsoft Teams will be designated for project discussion, Meetings are expected to be an hour. The expected response time of a message is 1-2 hours in order to maintain communication between team members. The amount of responses of a team memeber is a minimum of one if said member were to be mentioned (@mention) via Microsoft Teams.

Peer Feedback

Informal peer feedback will occur via Microsoft Teams, and will not be:

  • Disrespectful
  • Sexist/racist
  • Dismissive

Formal peer feedback will be completed via SparkPlus.

Use of Tools:

  • HTML 5: To be used in any prefered HTML editor of an individual's choice (eg. Visual Sudio Code) and all webpages that are created and/or edited are to be saved within GitHub

  • CSS 3: To be used in any prefered HTML editor of an individual's choice (eg. Visual Sudio Code) and all .css files that are created and/or edited are to be saved within GitHub in a "styles" folder

  • PHP: To be used in any prefered HTML editor of an individual's choice (eg. Visual Sudio Code) and all .php files that are created and/or edited are to be saved within GitHub in a "php" folder

  • MySQL 8.0: all ".sql" files that are created and/or edited are to be saved within GitHub in a sql folder

Un-met expectations/Problems:

If a team member undergoes time restraints due to man-power, underestimated high work-time or insufficiant use of time, a notification to the team is required and immediate. Team members who are able to provide assistance towards said problem are to be creditited. If however, all of the members cannot provide immediate assistance to said team-member, they must email an RMIT tutor for assistance
